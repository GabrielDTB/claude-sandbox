# Use slirp4netns with --disable-host-loopback

Rootless podman's older network backend. The `--disable-host-loopback`
flag satisfies [[7]], but [[8]] is not addressed: LAN, RFC1918, CGNAT,
and metadata destinations all remain routable through the slirp stack.

Closing that gap requires layered firewall rules inside the container's
netns, which re-introduces the per-address maintenance surface we
wanted to avoid and is fragile against routing changes on the host.

Superseded by [[10]], whose isolation model blocks the broader family
by default rather than by allowlist.
