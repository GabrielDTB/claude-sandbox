# Sandbox presents stub credentials to Claude Code

Claude Code refuses to start without credentials in its config. If
the sandbox contained no credential material at all, the agent
wouldn't launch. We write a syntactically-valid but non-functional
stub into the sandbox so Claude Code boots, and rely on [[3]] to
translate its requests into authenticated ones out-of-band.

The stub value is rewritten on every launch, so a compromised agent
that overwrites it with a real token stolen from elsewhere cannot
have that token persist across sessions via
[[persist-state-across-sessions]]. The stub's placeholder string is
deliberately recognizable (e.g. a sentinel prefix) so a misrouted
request landing at the real API fails fast rather than looking like
an auth error from a real-but-wrong token.
