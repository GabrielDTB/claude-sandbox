There are currently 4 major features I want to add to the sandbox.

1. Inheriting globals like skills and memories.
   These need to be scoped (don't want all skills and memories to go to all sandboxed).
   Also, the ability for these things to propagate back out from the sandboxes would be nice.
   Often I'll write a skill inside a project and want to factor it out for general use.
   Of course, there needs to be a review layer by the user.
   Nothing should be written back to a global location without user review.
   This goes into...

2. Enabling manual user review of changes to files in scope of a sandbox.
   Currently, I just run lazygit and look at the diff.
   But this doesn't track files in locations outside of the workspace directory.
   I want to track *all* changes and give an interface to review them.

3. Allowing multiple sandbox instances to work on the same project in a sane way.
   This probably consists of essentially creating two copies of the project, making changes in both, and figuring out how to merge them.
   This is what git is made for, but not everything is git tracked, and we don't want to make the copies manually.

4. Persistent daemon mode.
   There should be some mechanism to launch a sandbox which can be interacted with persistently, separate from Claude Code.
   This would allow for eg. Dispatch-like capabilities to be built on top.
