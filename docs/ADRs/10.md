# Use pasta with --no-map-gw --map-guest-addr none

Satisfies [[7]] and [[8]] via pasta's stricter default isolation
model. `--no-map-gw` prevents the container from using the host as
its default gateway, blocking "talk to the host as a router" attacks.
`--map-guest-addr none` prevents the container's guest IP from being
spoofable onto host-reachable interfaces.

Preferred over [[9]] because the isolation is the default rather than
layered on via supplementary firewall rules, and pasta is podman's
current recommended rootless network backend (slirp4netns is in
maintenance mode).

The [[3]]-reachability punchthrough is a separate concern, handled
differently for embedded and external proxy deployments — see notes
downstream.
