# Inheriting globals like skills and memories

Currently, each sandboxed project has its own skills and memories.
However, these often relate to the foundational technologies of the project.
Many projects may work with these technologies, and so a great deal of time is wasted either reproducing these, manually coping them over, or simply never taking advantage of them.

We should, at the minimum, have some cental location to store skills, memories, and data from related features, and be able to pass them into individual sandboxes.
They should be scoped, since we don't want all skills and memories to be present for all projects, and composable, so that you don't have to enumerate every single one.

For mechanism, I'm thinking tags, with directories assigning tags implicitly by directory name
(eg. `/global/path/to/root/dir/./skills/languages/python/typing.md` <- typing.md would be a skill, implicitly with the tag `languages/python`.
Then, profiles could also be defined in terms of tags and individual files somewhere in the config.
Passing in by profile, tag, or individual file should be possible.

Profiles should be declared in `$XDG_CONFIG_HOME/<dir>/config.toml`.
The actual skills and memories should be in `$XDG_DATA_HOME/<dir>/skills` / `$XDG_DATA_HOME/<dir>/memory`

The profile schema should look like:

```toml
[profiles.<name>]
# shared tags
tags = ["<tag name>", ...]

[profiles.<name>.skills]
# skill specific tags and files
tags = ["<tag name>", ...]
extra_files = ["relative path within skills directory", ...]

[profiles.<name>.memory]
# memory specific tags and files
tags = ["<tag name>", ...]
extra_files = ["relative path within memory directory", ...]
```

Tag matching is prefix.
So `languages` matches `languages/python`.

To get skills tags, the shared tags are simply unioned with the skills specific tags.
Same thing with memory.

CLI flags for selecting these should be like:
```
--profile <profile>
--skill-tag <tag> # repeatable, adds tag to skill matcher
--memory-tag <tag>
--skill-file <path> # repeatable, add specific skill file
--memory-file <path>
```

Skills and memories are bind-mounted read-only to the sandbox user's .claude directory.
