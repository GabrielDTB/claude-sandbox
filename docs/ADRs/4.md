# Proxy runs outside the agent's sandbox

Follows from [[3]]: if the proxy held real credentials but lived inside
the agent's sandbox, the agent could read them directly and [[1]] is
unsolved. The proxy process must be in a trust boundary the agent
cannot cross.

Concretely, this means one of: a sibling container under a different
UID with no shared mounts; a systemd service on the host under a
dedicated service account; or a separate machine reachable over the
network. All three are in scope and chosen per deployment.
