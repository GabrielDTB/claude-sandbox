# Sandbox cannot reach host LAN / cloud metadata / VPN

Same family as [[7]] but broader. RFC1918 (10/8, 172.16/12,
192.168/16), CGNAT (100.64/10), link-local (169.254/16), and
tailnet ranges are all trust-adjacent to the host and must not be
reachable from the sandbox.

Cloud metadata at 169.254.169.254 is the most sensitive single
endpoint: on AWS/GCP/Azure it serves instance credentials over
unauthenticated HTTP to any local caller. A sandbox that can
reach it inherits the host VM's IAM role.

Exception: the proxy ([[3]]) may legitimately live on one of these
ranges — a shared team proxy on a Tailscale address, for example.
That single resolved destination is punched through the block;
everything else in the family remains unreachable.
