# Proxy must not be world-accessible

Follows from [[3]]: the proxy speaks Claude Code's API with real
credentials on the far side. Any caller that reaches it gets free
use of those credentials — the proxy is the credential, laundered
through a network endpoint.

"Not world-accessible" means the proxy's listening socket must be
reachable only by the intended sandbox. When the proxy and sandbox
share a host, loopback binding with pasta port-forwarding into the
sandbox is sufficient. When they are on different hosts, the link
must be authenticated (shared secret header, mTLS, or network-level
restriction like a Tailscale ACL) — a raw TCP listener on a LAN
address is not acceptable.
