# Sandbox cannot reach host loopback

Independent of [[1]]: even with credentials solved, the sandbox
sitting on the host's loopback interface can reach any service
the user runs locally. Concrete targets worth enumerating: local
databases (Postgres on 5432, Redis on 6379), dev HTTP servers
with no auth, an SSH agent proxied over TCP, Docker's API on
2375/2376, editor language servers, and any other "trust localhost"
service the user runs.

Most of these grant arbitrary code execution or data exfiltration
to any loopback caller. The agent must not be able to address
`127.0.0.1` or `::1` and land on the host.
